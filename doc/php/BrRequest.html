BrRequest helper to work with request

extends BrObject

Helper to work with request.

There is one global request object (singleton) accessible via br()->request(). You can still create own request obejct if needed by $request = new BrRequest();

Again, if you are working with global config object you can use br()->config(name, defaultValue) instead of br()->config()->get(name, defaultValue) for simplicity.

br()
  ->request()
    ->route('/index.html', function() {
        // display homepage
      })
    ->route('/order.html?client=([0-9]+)', function($matches) {
        // display order of client $matches[1]
      })
    ->routeGET('/pay.html', function() {
        // display payment page
      })
    ->routePOST('/pay.html', function() {
        // handle POST to pay.html
      })
    ->routeIndex(function() {
        // handle request to root url of your site
      })
    ->routeDefault()
;

Once some of the routes processed - other ones will be skipped. Simple, isn't it?
